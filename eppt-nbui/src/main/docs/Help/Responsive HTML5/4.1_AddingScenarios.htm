4.1. Adding Scenarios

Clicking “Add” in the Scenarios panel will open the following window shown in Figure 4-2.

 

Figure 4-2. New Scenario Run Window.

 

A default Scenario Name (1) is provided but can be modified. A Description can also be added (2) (optional). For this tutorial, CalSim 2 is the Model (3). CalLite and CalSim 3 models may also be loaded.

Study Main directory refers to the folder in which the study is located. For example, Figure 4-3 shows the example file path which is “…\DCR2017_noCC_x64_original.”

Figure 4-3. Study Main Directory example location.

 

Figure 4-4 shows the Required Files. File paths for DV, SV, and INIT study files to be designated. The QA_QC file will be generated after running the WRESL Script, described in Appendix C from the EPPT Appendices. The QA_QC file name and location can be chosen.

Alias Names are short descriptors for the required files. Lastly, the DSS A Part and F Part are to be defined. The A Part ranges from CALSIM to CALSIM_CYCLE1 to CALSIM_CYCLE13 The DSS F Part is typically “2020D09E.”.  This can be verified by opening the DSS files with HEC-DSSVue.

Figure 4-4. Required Files.

 

The last section of the Edit Scenario Run window relates to the QA/QC Report shown in Figure 4-5.

The EPPT WRESL Script can be left unchanged. The Water Year Table file should refer to the study “wytypes.table” file path.

Figure 4-5. QA-QC Report.

 

After completing the fields, the Scenario Run window should look similar to Figure 4-6.

 

Figure 4-6. Filled out Scenario Run window.

 

For more detailed description of the Scenarios window, see Appendix B in the EPPT Appendices.
